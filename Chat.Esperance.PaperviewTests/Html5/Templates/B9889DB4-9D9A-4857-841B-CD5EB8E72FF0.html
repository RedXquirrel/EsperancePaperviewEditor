To Create a Presentation Template from this existing Microformat:

The Paperview editor creates a separation of concerns between the Microformat Data Layer (the JSON that it creates for each new document), and the Presntation Layer - that is, how the generic Microformat JSON is rendered.

This means that for any Paperview Document Type - which we call a Paperview Microformat - different Presentation Templates can be created to render the same data: we call the data 'artifacts' as each piece of data could be text, and image, and so forth.

The following process shows you how to create a different presentation rendering of this Paperview Document Type - you will be creating a Microformat Presentation Template for this Microformat (b9889db4-9d9a-4857-841b-cd5eb8e72ff0 (Image Album)).

Any users who obtain this Paperview Presentation Template will be able to view, and interact with, this 'b9889db4-9d9a-4857-841b-cd5eb8e72ff0 (Image Album)' Microformat, in the manner that you code how the JSON is manipulated in it's HTML5 page.

  2. Change the Guid of the Microformat Id ("mfid":"b9889db4-9d9a-4857-841b-cd5eb8e72ff0") to a new, original Guid - eg. "mfid":"71d5155b-75f4-4636-822c-05c7709f7ef5". This is the Guid that identifies YOUR Presentation template of this Microformat; the microformat itself is identified by the Microformat Derivation property. So by way of example, although the Microformat Id of this exact document that you are looking at is b9889db4-9d9a-4857-841b-cd5eb8e72ff0, this is merely a Presentation Format Id - the Microformat itself is specified in the Microformat Derivation property.
  3. Change the Guid of the Document Id ({"document":{"id":"a5c24278-2c0d-47a2-bddf-ea3d2fd34768", ...) to a new, original Guid - eg. {"document":{"id":"9594756b-ad0b-4767-b61d-5fc1e7a942c4", ... }. As you are creating this as a template, this is merely arbitrary becaue when a user uses the Paperview Editor to create a new Paperview document using your Paperview Presentation Template, the editor will insert a new, unique Guid, in each new document that is created using your new Paperview Presentation Template.
  4. Ensure that the Microformat Derivation ("mfderivation":"b9889db4-9d9a-4857-841b-cd5eb8e72ff0") is the one that you are deriving from (in this case, the Microformat Id specified above - b9889db4-9d9a-4857-841b-cd5eb8e72ff0).

    var esperancejson = JSON.parse(esperance);
  6. Change the html filename to reflect your new Microformat Id, in this example, to 71d5155b-75f4-4636-822c-05c7709f7ef5.html
  7. Using the above Microformat Property Definition table, utilise the following columns to determine your preferred Javascript coding style: JSON Path, Javascript Path, Example Javascript Declaration, and Example Html5 Rendering.
  8. Once you have the full presesentation functionality working according to how you want to present the artifacts of the Microformat, ammend the script tag above to: 
        <script id="esperance">
        __paperview-template__
    </script>
  9. Import the new Paperview Presentation Template into your Paperview Editor and it will become an option when creating new Paperview pages using this Microformat (as defined in the JSON "mfderivation":"b9889db4-9d9a-4857-841b-cd5eb8e72ff0").
    Note: The editor groups all Presentation Templates together that have the same Microformat Derivation property, but uses the Microformat Id property to determine which of the html template files to select when the user chooses one of them!
  10. Keep in mind:
    • Please note that we plan releasing Paperview Viewers that will also allow the 'specialist viewer' of a document that has been created using your Presentation Microformat, to switch between bespoke Presentation Templates that they might have installed. Of course, if we have inserted Microformatted JSON into your Microformat Presentation Template, we can just as easily strip it out and re-render it in another bespoke Presentation Template. (Don't be alarmed by this as the long term revenue model for premium content is hoped to include the publisher of documents that were created using the original Presentation Template that the document was originally 'published' under.)
    • So remember, where "Content is King", how it is presented is Queen!
    • So we encourage you to make Microformat Presentation Templates that are truely queen to the generic Microformat artifacts that your templates will be rendering!
    • If your Presentation Template is so good, viewers will have no need to select another!
    • We do think, however, that the majority of viewers will look at documents using regulkar browsers (etc), so you can (in the first instance) consider your Presentation Template to be the default viewing template for Paperview documents that are viewed using the ilk of regular browsers!
